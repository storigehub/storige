# 🤖 AUTOPILOT: 자동 오케스트레이션 모드

> 이 파일을 Claude Code에 로드하면, 에이전트들이 자동으로 일을 넘기며 처리하고,
> **오너(Papas님)의 판단이 필요한 순간에만 멈춰서 물어봅니다.**

---

## 사용법

Claude Code에서 이렇게 시작합니다:

```
CLAUDE.md를 읽고, agents/AUTOPILOT.md를 로드해.
Phase [번호] Sprint [번호]를 자동 파이프라인으로 실행해줘.
```

또는 더 간단하게:

```
오토파일럿 모드로 Phase 1을 시작해.
```

---

## 자동 파이프라인 규칙

### 핵심 원칙

```
1. 에이전트 역할 전환은 자동으로 수행한다
2. 각 단계의 결과를 다음 에이전트에게 자동으로 전달한다
3. 오너에게 묻지 않고 진행할 수 있는 일은 자동 진행한다
4. 아래 [STOP] 조건에 해당하면 반드시 멈추고 오너에게 보고한다
5. 오너가 승인/지시하면 다시 자동으로 이어서 진행한다
```

### [STOP] 오너에게 반드시 물어야 하는 순간

```
🔴 STOP-1: Phase/Sprint 시작 전
   → "Phase X Sprint Y를 시작합니다. 태스크 목록은 이렇습니다. 진행할까요?"

🔴 STOP-2: 기술적 선택지가 2개 이상일 때
   → "A 방법과 B 방법이 있습니다. 트레이드오프는 이렇습니다. 어떤 걸로 할까요?"

🔴 STOP-3: 코드리뷰에서 Critical 이슈 발견 시
   → "보안 문제를 발견했습니다. [내용]. 수정 방향을 확인해주세요."

🔴 STOP-4: QC 검증에서 실패 항목 있을 때
   → "Phase 완료 검증 결과: N건 실패. 상세 내용 확인하고 진행 방향을 결정해주세요."

🔴 STOP-5: Phase 완료 → Go/No-Go 판정
   → "Phase X 완료. 결과 요약은 이렇습니다. 다음 Phase로 넘어갈까요?"

🔴 STOP-6: 예상 밖의 에러/문제 발생
   → "예상치 못한 문제가 발생했습니다. [상황 설명]. 어떻게 처리할까요?"

🔴 STOP-7: 스코프 변경이 필요할 때
   → "이 기능을 구현하려면 당초 계획보다 시간이 더 필요합니다. 스코프를 조정할까요?"
```

### [AUTO] 자동으로 진행하는 일

```
✅ AUTO-1: 테스트 작성 (테스터 역할)
✅ AUTO-2: 코드 구현 (CTO 역할)
✅ AUTO-3: 코드리뷰 Minor/Major 이슈 자동 수정
✅ AUTO-4: 디자인 사양 적용 (디자인팀장 참조)
✅ AUTO-5: 리팩토링 (200줄 초과 파일 분리 등)
✅ AUTO-6: 테스트 실행 및 결과 보고
✅ AUTO-7: 커밋 메시지 작성 및 파일 저장
```

## 오너의 응답 방법

### 간단한 승인
```
Go
```
또는
```
ㅇㅇ 진행해
```

### 선택지 결정
```
A로 해
```

### 수정 요청
```
B-3 아코디언에서 사진이 3장 이상이면 가로 스크롤 되게 해줘.
나머지는 Go.
```

### 방향 변경
```
시크릿 코드 UI를 좀 더 어둡게 해줘. 보안 느낌이 나게.
디자인팀장한테 톤 재조정 시키고 나서 진행해.
```

### 진행 상황 확인
```
지금 어디까지 했어?
```

### 일시 중지
```
잠깐 멈춰. 내가 다시 시작하라고 할 때까지 대기.
```

## 컨텍스트 유지: 세션이 끊겼을 때

Claude Code 세션이 끊기면 컨텍스트가 사라집니다. 이를 대비해 매 Sprint 완료 시 상태를 CLAUDE.md에 자동 기록합니다:

```markdown
## 현재 진행 상태 (자동 업데이트)

최종 업데이트: 2026-04-05 15:30
현재 Phase: 2
현재 Sprint: 2-3
완료 이슈: B-1, B-2, B-3, B-4
진행 중: B-5 (미디어 뷰 + 지도 뷰)
블로커: 없음
다음 STOP 예정: Sprint 2 완료 후 QC 검증
```

새 세션을 시작할 때:

```
CLAUDE.md를 읽고 오토파일럿 모드로 이어서 진행해.
```

이렇게 하면 마지막 상태부터 자동으로 이어갑니다.

## 요약: 오너가 하는 일

```
1. "오토파일럿으로 Phase X 시작해" → 자동 진행
2. [STOP]에서 멈추면 → 읽고 판단 ("A로 해", "Go", "이거 수정해")
3. 궁금하면 → "지금 어디야?" 물어보기
4. 세션 끊기면 → "이어서 진행해" 한마디
```

전체 개발 중 오너가 직접 판단하는 횟수:
  Phase 1: 약 3~5회
  Phase 2: 약 5~8회
  Phase 3: 약 8~12회 (보안 결정 多)
  Phase 4: 약 5~8회
  Phase 5: 약 3~5회 (최종 릴리스 판정 포함)
