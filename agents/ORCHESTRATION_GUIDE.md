## 에이전트 구성

| 에이전트 | 파일 | 핵심 역할 | 호출 시점 |
|---------|------|----------|----------|
| 🎯 CEO | `AGENT_CEO.md` | 비전 수호, 우선순위, Go/No-Go | Phase 시작/완료, 기능 충돌 시 |
| 🏗️ CTO | `AGENT_CTO.md` | 아키텍처, 기술 결정, 작업 분배 | 매 세션, 오케스트레이션 |
| 🔍 코드리뷰어 | `AGENT_CODE_REVIEWER.md` | 코드 품질, 보안, 표준 | 코드 작성 후 |
| 🎨 디자인팀장 | `AGENT_DESIGN_LEAD.md` | UI/UX, 디자인 시스템, 접근성 | UI 구현 전/후 |
| ✅ QC | `AGENT_QC.md` | 통합 품질, 시나리오 검증, 릴리스 | Phase 완료 시 |
| 🧪 테스터 | `AGENT_TESTER.md` | TDD, 테스트 코드, 커버리지 | 기능 구현 전/후 |

## Claude Code에서 사용하는 방법

### 방법 1: 세션별 역할 지정 (권장)

Claude Code 세션을 시작할 때, 해당 세션의 역할을 지정합니다:

```bash
# 터미널에서 Claude Code 시작 시
claude

# Claude Code 프롬프트에서:
> @agents/AGENT_CTO.md 를 읽고 CTO 역할로 Phase 1-1 작업을 분배해줘.
```

또는 더 간단하게:

```
> CTO 역할이야. AGENT_CTO.md를 참조해. Phase 1의 스프린트를 태스크로 분해하고 작업 순서를 정해줘.
```

### 방법 2: 멀티 에이전트 대화 (한 세션에서 역할 전환)

```
> [CTO 모드] Phase 2-1 일기 에디터 구현을 시작한다.
> 먼저 [테스터 모드]로 전환해서 useDiary 훅의 테스트를 먼저 작성해줘.
> (테스트 작성 완료)
> [CTO 모드]로 돌아와서 이 테스트를 통과하는 코드를 구현해줘.
> (코드 작성 완료)
> [코드리뷰어 모드]로 전환해서 방금 작성한 코드를 리뷰해줘.
> (리뷰 완료, 수정사항 반영)
> [디자인팀장 모드]로 전환해서 DiaryListView UI를 검수해줘.
```

### 방법 3: CLAUDE.md에 현재 역할 기록

프로젝트 루트의 `CLAUDE.md` 하단에 현재 세션 컨텍스트를 추가:

```markdown
## 현재 세션
- Phase: 2
- Sprint: 2-1
- 활성 에이전트: CTO + 테스터
- 진행 중 태스크: B-1 (일기 작성 에디터)
```

```

                ▼
┌─ 테스터 ──────────────────────────────────────┐

                ▼
┌─ CTO ─────────────────────────────────────────┐

                ▼
┌─ 디자인팀장 ──────────────────────────────────┐
```

## Phase 1 시작 예시 (실전 가이드)

### Step 1: CTO로 시작

```
당신은 Storige 프로젝트의 CTO입니다.
agents/AGENT_CTO.md를 읽고 역할을 수행하세요.
CLAUDE.md와 STORIGE_DEV_PLAN.md를 참조하세요.

Phase 1-1 (프로젝트 초기화)을 시작합니다.
태스크를 분해하고 순서대로 실행해주세요.
```

### Step 2: 테스터로 전환

```
이제 테스터 역할로 전환합니다.
agents/AGENT_TESTER.md를 참조하세요.

Phase 1-3 Auth 모듈의 테스트를 먼저 작성해주세요.
STORIGE_DEV_PLAN.md 섹션 8.1 (Auth QA)을 기반으로 합니다.
```

### Step 3: CTO로 복귀해서 구현

```
CTO 역할로 복귀합니다.
테스터가 작성한 Auth 테스트를 통과하는 코드를 구현해주세요.
TDD 원칙: Green → Refactor
```

### Step 4: 코드리뷰어로 전환

```
코드리뷰어 역할로 전환합니다.
agents/AGENT_CODE_REVIEWER.md를 참조하세요.

방금 구현한 Auth 모듈 코드를 리뷰해주세요.
특히 보안 체크리스트를 중점 검토하세요.
```

### Step 5: 디자인팀장으로 전환

```
디자인팀장 역할로 전환합니다.
agents/AGENT_DESIGN_LEAD.md를 참조하세요.

Phase 1-4 기본 레이아웃(Header, Nav, FAB)의 UI 사양을 제공해주세요.
docs/storige-prototype.html을 기준으로 합니다.
```

## 에이전트 간 커뮤니케이션 규칙

### 형식
```
[발신 에이전트 → 수신 에이전트] 요청/보고/질문
━━━━━━━━━━━━━━━━━━━━━━━
내용
━━━━━━━━━━━━━━━━━━━━━━━
```

### 에스컬레이션 경로
```
일반 이슈:    테스터 → QC → CTO
보안 이슈:    코드리뷰어 → CTO (즉시)
디자인 충돌:  디자인팀장 → CTO → CEO
우선순위 충돌: CTO → CEO
릴리스 판정:  QC → CEO
```

### 블로킹 이슈 처리
```
1. 블로킹 발견 → 즉시 CTO에게 보고
2. CTO가 영향 범위 판단
3. 우회 가능 → 우회 + 기술 부채 등록
4. 우회 불가 → CEO에게 일정 영향 보고
5. CEO가 스코프 조정 또는 일정 변경 결정
```
