## 3. AGENT_CTO가 알아야 할 블로커·후속

1. **Supabase**  
   - 프로덕션 DB에 `mystory_sessions` 마이그레이션 **적용 여부** 확인. 미적용 시 `/mystory` 런타임 오류.
2. **Vercel 프로젝트 Node 버전**  
   - Actions는 Node **22**로 CLI 실행. **실제 `next build` Node**는 Vercel Dashboard → Project → Node.js Version에서 설정. CI와 맞출지 제품 결정.
3. **로컬 빌드 경고**  
   - 상위 디렉터리 `package-lock.json` 때문에 Next가 workspace root를 잘못 추론하는 경고 가능 — [`next.config.ts`의 `turbopack.root`](https://nextjs.org/docs/app/api-reference/config/next-config-js/turbopack) 또는 중복 lockfile 정리 검토.
4. **별도 나의이야기 레포**  
   - Storige 변경은 **다른 레포를 수정하지 않음**. 질문 풀은 정적 이관 — 원본과 **자동 동기화 없음**.

## 4. 리밋·세션 중단 시 “터미널에서 이어하기”

리밋으로 **중간에 끊긴 터미널 작업**이 있으면 아래 순서로 재개한다.

### 4.1 항상 먼저

```bash
cd /Users/yohan/Documents/claude/storige
git fetch origin && git status
git log -3 --oneline
```

원격이 앞서 있으면: `git pull origin main`

### 4.2 배포·CI 확인 (끊기기 직전에 `npm run build` / `gh` 돌리던 경우)

```bash
npm install
npm run build
gh run list --repo storigehub/storige --limit 5
# 특정 런 로그: gh run view <run-id> --log-failed
```

### 4.3 ESLint (경고 21건 정리를 이어할 때)

```bash
npx eslint . --max-warnings 999
```

- **아직 안 한 작업:** unused 제거, `<img>` → `next/image`, `layout.tsx` 폰트 규칙, `SecretCredentialTable` unused-expressions 등. 에러는 이미 0.

### 4.4 Supabase CLI / 마이그레이션 (대시보드 SQL 실행 전 단계에서 끊긴 경우)

- 마이그레이션 파일: `supabase/migrations/20260409120000_mystory_sessions.sql`
- 로컬에 Supabase CLI가 없으면 **Dashboard → SQL**에 파일 내용 붙여 실행하는 방식으로 동일 결과.

### 4.5 Claude Code `cc` 등 대화형 세션

- 터미널에 `cc`만 떠 있고 **작업 내용이 대화에만 있으면** 복구 불가. **이 문서 + `git log` + 변경 파일**을 기준으로 요청을 다시 내리면 된다.
