## 보유 자산 (구현 재개 시 바로 사용 가능)

```
DB (프로덕션 적용 완료):
  memorial_pages: slug, title, bio, cover_image, is_public, birth_date, death_date
  memorial_messages: author_name, content, relation, is_approved

훅 (완성):
  src/hooks/useMemorial.ts
  - useMyMemorial()       — 내 추모관 생성/편집
  - usePublicMemorial()   — 공개 페이지 조회 + 방명록 작성
  - useMemorialAdmin()    — 방명록 승인/거절/삭제

없는 것: 페이지와 컴포넌트 전체
```

---

## 기획 방향 — "기억 축제"

> 침울한 장례 페이지가 아니라, 그 사람의 삶을 따뜻하게 기념하는 공간

| 기존 추모관 보편 느낌 | Storige 추모관 |
|---------------------|----------------|
| 검은 배경, 촛불 아이콘 | 따뜻한 아이보리/크림 배경 |
| "삼가 고인의 명복을" | "기억해주셔서 감사합니다" |
| 방명록 | "기억을 남기다" |
| 죽음 중심 | **삶 중심** — 생일, 좋아했던 것, 남긴 말 |

## 화면 구성 (2개 페이지)

### ① 공개 추모 페이지 `/memorial/[slug]` — 비로그인 접근 가능

```
┌─────────────────────────────────────────┐
│  [커버 사진 풀블리드 — 따뜻한 그라디언트 오버레이]  │
│                                         │
│  홍길동 님을 기억합니다                     │
│  1945. 03. 15 — 2024. 11. 22           │
└─────────────────────────────────────────┘

│ 그분에 대하여 (bio)
│ "평생 가족을 위해 살았던, 작은 것에도 기뻐할 줄 알던 분"

│ 기억을 남겨주세요  ← 방명록 작성 폼
│   이름 / 관계 / 메시지 → 오너 승인 후 공개

│ 남겨진 기억들  ← 승인된 방명록 카드들

│ QR 코드 (장례식장 인쇄용)
```

### ② 추모관 설정 `/settings/memorial` — 로그인 필요

```
• 추모관 생성/편집 (커버 사진, 이름, 생몰년월일, 소개글)
• 공개/비공개 토글
• 방명록 승인 관리 (대기 중 / 공개 / 거절)
• 링크 복사 + QR 코드 다운로드
```

## 컬러 톤 — Midnight Archive에서 살짝 이탈

기존 앱의 차가운 코발트 계열을 유지하되, 추모관만 따뜻한 amber 악센트 적용:

```css
추모관 포인트 컬러: #B45309  (amber-700 — 따뜻한 호박색)
배경:             #FEFAF5  (크림 화이트)
커버 그라디언트:   warm amber → transparent (검은색 대신)
방명록 카드:       bg-white, 왼쪽 보더 amber
```

## 구현 범위 (MVP)

```
필수 (구현 재개 시):
  - /memorial/[slug] 공개 페이지
  - /settings/memorial 관리 화면
  - 방명록 작성 + 승인 플로우
  - QR 코드 생성 (링크 복사 fallback)

이후 고도화:
  - 음악 재생 (추모 음악)
  - 사진 갤러리 (현재 cover_image 1장)
  - remember.storige.co.kr/[slug] 서브도메인 연결
```

## 미결 결정사항 (구현 재개 전 오너 확인 필요)

**추모관을 누가 만드나?**

```
A) 본인이 생전에 직접 만들어 두는 방식
   → "나의 추모관 미리 만들기" — 유고 기능과 자연스럽게 연결
   → 설정에서 "유고 시 공개" 토글 추가 가능

B) 가족이 사후에 대신 만드는 방식
   → 별도 계정 없이 방문자가 slug로 접근
   → 오너 = 생존 가족 (별도 계정)

C) 둘 다 가능 (생전 예약 + 가족 대리 생성)
```
